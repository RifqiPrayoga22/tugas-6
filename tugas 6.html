Slide 2

Pengantar Generic

Apa itu Generic? Fitur untuk membuat kode yang bekerja dengan berbagai tipe data. Analog: Kotak penyimpanan yang bisa menampung apa saja, tapi dengan label tipe. Manfaat: Type safety (keamanan tipe), Code reusability (penggunaan ulang), Compile-time checking.

Slide 3

Masalah Tanpa Generic

class Box { dynamic data; // Bisa apa saja Box(this.data); } void main() { var box = Box(42); int value = box.data; // Runtime error jika bukan int // Tidak aman! }

Tanpa generic, pengembang sering menggunakan dynamic agar kotak bisa menampung nilai apapun. Konsekuensinya, kesalahan tipe baru diketahui saat program berjalan (runtime error), bukan saat kompilasi. Hal ini membuat kode menjadi tidak aman dan rawan bug.

Slide 4

Solusi dengan Generic

class Box<T> { T data; Box(this.data); } void main() { var intBox = Box<int>(42); int value = intBox.data; // Aman, terjamin int }

Dengan menambahkan parameter tipe <T> pada kelas Box, compiler tahu tipe data apa yang akan disimpan. Saat membuat Box<int>(42), compiler memastikan data selalu bertipe int. Error akan muncul saat kompilasi, bukan runtime.

Slide 5

contoh Sintaks Generic pada class dan function

class MyClass<T> { T data; MyClass(this.data); T getData() { return data; } } T myFunc<T>(T value) { return value; } void main() { // Class generic var myIntClass = MyClass<int>(10); var myStringClass = MyClass<String>("test"); print(myIntClass.getData()); // Output: 10 print(myStringClass.getData()); // Output: test // Fungsi generic var intValue = myFunc<int>(100); var stringValue = myFunc<String>("hello"); print(intValue); // Output: 100 print(stringValue); // Output: hello }

Parameter tipe <T> berfungsi sebagai placeholder yang digantikan dengan tipe konkret saat kelas atau fungsi digunakan. Contoh MyClass<int>(10) mengubah semua T menjadi int. Satu definisi kelas dapat digunakan berulang untuk tipe data berbeda.

Slide 6

contoh Kelas Box<T>

class Box<T> { T? _value; // Tipe data T, nullable Box(this._value); // Constructor T? get value => _value; // Getter void setValue(T value) { _value = value; } } void main() { // Menggunakan Box dengan tipe data int var intBox = Box<int>(10); print('Int Box Value: ${intBox.value}'); intBox.setValue(20); print('Int Box Value After Set: ${intBox.value}'); // Menggunakan Box dengan tipe data String var stringBox = Box<String>('Hello, Generic!'); print('String Box Value: ${stringBox.value}'); stringBox.setValue('Hello World'); print('String Box Value After Set: ${stringBox.value}'); // Menggunakan Box dengan tipe data double var doubleBox = Box<double>(3.14); print('Double Box Value: ${doubleBox.value}'); }

Kelas Box<T> menyimpan data bertipe T tanpa peduli apakah itu integer, string, atau double. Getter dan setter menjaga konsistensi tipe. Saat membuat Box<int>(10), compiler memastikan semua operasi hanya menerima integer.

Slide 7

contoh Kelas Pair<K, V>

class Pair<K, V> { K key; V value; Pair(this.key, this.value); void printPair() { print('Key: $key, Value: $value'); } } void main() { // Pair dengan tipe data String dan int var pair1 = Pair<String, int>('age', 30); pair1.printPair(); // Output: Key: age, Value: 30 // Pair dengan tipe data String dan String var pair2 = Pair<String, String>('name', 'Alice'); pair2.printPair(); // Output: Key: name, Value: Alice // Pair dengan tipe data int dan double var pair3 = Pair<int, double>(1, 2.71); pair3.printPair(); // Output: Key: 1, Value: 2.71 }

Generic dengan dua parameter tipe Pair<K, V> memungkinkan penyimpanan dua tipe berbeda dalam satu struktur data. Berguna untuk pasangan kunci-nilai, koordinat, atau kombinasi data lainnya.

Slide 8

menggabungkan dua nilai dengan tipe data yang berbeda

String combine<T, U>(T value1, U value2) { return '$value1, $value2'; } void main() { var result1 = combine<String, int>('Number is ', 10); print(result1); // Output: Number is , 10 var result2 = combine<int, double>(10, 20.5); print(result2); // Output: 10, 20.5 }

Fungsi generic dengan dua parameter tipe memungkinkan penggabungan dua nilai berbeda tipe menjadi satu string. T dan U bisa berupa tipe apa saja.

Slide 9

contoh Penggunaan Pair<K, V>

void main() { var pair1 = Pair<String, int>('age', 30); pair1.printPair(); // Output: Key: age, Value: 30 var pair2 = Pair<String, String>('name', 'Alice'); pair2.printPair(); // Output: Key: name, Value: Alice var pair3 = Pair<int, double>(1, 2.71); pair3.printPair(); // Output: Key: 1, Value: 2.71 }

Slide ini menunjukkan bagaimana Pair<K, V> digunakan dalam praktik dengan berbagai kombinasi tipe data. Satu kelas Pair dapat menangani pasangan String-int, String-String, maupun int-double.

Slide 10

contoh Penggunaan combine<T, U>

void main() { var result1 = combine<String, int>('Number is ', 10); print(result1); // Output: Number is , 10 var result2 = combine<int, double>(10, 20.5); print(result2); // Output: 10, 20.5 }

Demonstrasi pemanggilan fungsi combine dengan berbagai tipe parameter. Fungsi ini menggabungkan String dengan int, maupun int dengan double, menunjukkan fleksibilitas generic.

Slide 11

Pembatasan Tipe dan manfaatnya

Keamanan Tipe yang Lebih Baik

Memastikan bahwa kode hanya bekerja dengan tipe data yang kompatibel, sehingga mengurangi risiko runtime error.

Peningkatan Keterbacaan Kode

Membuat kode lebih jelas dan mudah dibaca karena kita dapat melihat tipe data yang diharapkan secara eksplisit.

Memungkinkan Penggunaan Method dari Tipe Data Tertentu

Memungkinkan kita untuk memanggil method yang spesifik untuk tipe data yang dibatasi.

Mencegah Penggunaan Tipe Data yang Tidak Sesuai

Mencegah penggunaan tipe data yang tidak sesuai dengan logika kode.

Membantu dalam Desain dan Implementasi

Membantu kita dalam merancang dan mengimplementasikan kelas dan fungsi generic dengan lebih terstruktur dan terdefinisi.

Slide 12

Pembatasan Tipe pada Interface dan Class

void printIterable<T extends Iterable<dynamic>>(T iterable) { for (var item in iterable) { print(item); } } void main() { printIterable<List<int>>([1, 2, 3]); printIterable<Set<String>>({'apple', 'banana'}); // printIterable<int>(10); // Error: 'int' is not a subtype of 'Iterable' }

Dengan T extends Iterable, kita membatasi T hanya untuk tipe turunan Iterable (List, Set). Percobaan dengan int menghasilkan error kompilasi.

Slide 13

Pembatasan Tipe - Lanjutan

Pembatasan tipe tidak hanya pada Iterable. Kita bisa menggunakan class atau interface apapun sebagai batasan, misalnya T extends Comparable untuk memastikan tipe bisa dibandingkan, atau T extends Exception untuk turunan Exception.

// Contoh lain pembatasan tipe: class SortedBox<T extends Comparable> { T data; SortedBox(this.data); int compareTo(SortedBox<T> other) => data.compareTo(other.data); }
Slide 14

Generic vs Dynamic vs Object

AspectGeneric<T>dynamicObject
Type SafetyCompile-timeRuntimeCompile-time
FlexibilityTerbatas (sesuai T)Sangat fleksibelTerbatas
PerformanceOptimalOverheadOptimal
CastingTidak perluTidak perluPerlu casting
// Contoh perbedaan Box<int> box1 = Box<int>(42); // Generic: aman dynamic box2 = 42; // Dynamic: tidak aman Object box3 = 42; // Object: perlu casting

Generic<T> unggul dalam keamanan tipe dan performa. dynamic paling fleksibel tapi mengorbankan keamanan. Object aman namun perlu casting manual.

Slide 15

Contoh Cache<T>

class Cache<T> { final Map<String, T> _storage = {}; void set(String key, T value) { _storage[key] = value; } T? get(String key) { return _storage[key]; } void remove(String key) { _storage.remove(key); } } void main() { var userCache = Cache<String>(); userCache.set('name', 'Alice'); var productCache = Cache<Product>(); productCache.set('p1', Product('Laptop', 1000)); }

Cache adalah penyimpanan sementara berbasis kunci-nilai. Dengan generic, satu kelas Cache dapat menyimpan berbagai tipe data tanpa duplikasi kode, sambil tetap menjaga keamanan tipe.

Slide 16

Ringkasan & Latihan

Ringkasan:

  • Generic membuat kode type-safe dan reusable
  • Bisa digunakan di class, fungsi, collections
  • Type inference memudahkan penggunaan
  • Constraints membatasi tipe yang diperbolehkan

Latihan:

  • Buat class Stack<T> dengan method: push(), pop(), peek()
  • Buat fungsi mergeLists<T>(List<T> a, List<T> b) -> List<T>
  • Buat class Response<T> dengan field T? data dan String? error

Challenge:

  • Buat sistem repository generic
